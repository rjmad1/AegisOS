## 1. Data Classification and Taxonomy

AegisOS categorizes system and user data into distinct classes, each subject to specific lifecycle regulations:

* **Knowledge**: Source files, documentation index stores, and local RAG vector embeddings.
* **Telemetry**: OpenTelemetry trace spans, CPU/VRAM usage metrics, and network statistics.
* **Logs**: Structured JSON runtime application logs, HTTP access logs, and worker event history.
* **Evaluation Scorecards**: Model performance checks, prompt validation results, and grounding scores.
* **Engineering Memory**: History of previous developer chats, past agent decisions, and tool execution logs.
* **Artifacts**: Code outputs, file exports, and temporary workspace files generated by agent runs.
* **Backups**: Database backups, prompt templates, and configuration keys.
* **Audit Records**: Login histories, RBAC configuration audits, and security baseline logs.
* **Configuration**: Registry variables, model aliases, and active system settings.
* **Secrets**: DB credentials, model connection strings, mTLS certificate private keys.

## 2. Retention and Archival Schedules

Data assets are managed according to these retention and archival policies:

| Data Category | Retention (Hot Storage) | Archival (Cold Storage) | Deletion Method | Compliance |
|---|---|---|---|---|
| **Knowledge** | Indefinite (Active DB) | N/A | Manual purge / overwrite | Internal R&D |
| **Telemetry** | 14 Days | 30 Days (S3 Glacier/GCS) | Automated overwrite | Operational |
| **Logs** | 30 Days | 365 Days | Scripted deletion | SOC2 Auditing|
| **Evaluations** | 90 Days | 1 Year | Scripted deletion | ISO 27001 |
| **Memory** | 90 Days | 3 Years (Compressed) | Overwrite | GDPR/CCPA |
| **Artifacts** | 7 Days | N/A | Immediate filesystem delete| Clean Space |
| **Backups** | 30 Days (Daily rota) | 7 Years (Annual snapshots) | Cryptographic shredding | Business DR |
| **Audit Records**| 365 Days | 7 Years (Read-only WORM) | Cryptographic shredding | Regulatory |
| **Configuration**| Indefinite | N/A | Manual overwrite | Audit trace |
| **Secrets** | Ephemeral (Vault) | N/A | Zero-fill Memory wipe | Zero Trust |

---

## 3. Deletion and Cryptographic Shredding

To guarantee complete data destruction, AegisOS implements two deletion methods:
1. **Filesystem Zero-Fill Deletion**: Temporary workspaces (artifacts) are overwritten with random bytes before deletion, preventing recovery from physical SSD sectors.
2. **Cryptographic Shredding**: Backup files and audit logs are encrypted using separate keys. When a customer/user requests data deletion ("Right to be Forgotten" under GDPR), the specific key is deleted from the key store, making the corresponding archived data unreadable.

---

## 4. Compliance Audits & GDPR/HIPAA Controls

* **GDPR Compliance (Article 17)**: The platform provides SDK functions and REST APIs to query, export, and delete a user's entire chat history and local agent memory.
* **HIPAA Compliance**: For healthcare deployments, PII and PHI (Protected Health Information) data is automatically filtered and scrubbed by the Scranton Safety Firewall prior to RAG index caching or database logging.
* **Audit Trails**: Deletion actions are logged in the immutable audit log, documenting the timestamp, user ID, data range, and method of destruction.
