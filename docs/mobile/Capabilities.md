# AegisOS Mobile Command Center: Capability Matrix & Functional Decomposition

* **Governing Directive**: `SCOPE_REDUCTION_DIRECTIVE_V1.md`

This document details the Capability Matrix and Functional Decomposition for the AegisOS Mobile Command Center, scoped to the 8 approved thin-client modules.

---

## 1. Capability Matrix

| Module | Purpose | Priority | Dependencies | Mobile Responsibilities |
|---|---|---|---|---|
| **Executive Dashboard** | Real-time health overview of the connected AI infrastructure (GPU/CPU/RAM/disk, services, agents, models, queues, alerts). | P0 | WebSocket telemetry stream, REST status endpoint | Read-only visualization. No computation. |
| **AI Executive Chat** | Conversational interface forwarding messages to the workstation and streaming responses. | P0 | REST chat endpoint, WebSocket streaming | Message relay only. No local memory, RAG, context, or prompt engineering. |
| **Human Approval Center** | Biometric-gated approval queue for high-risk commands generated by the workstation. | P0 | REST commands endpoint, WebSocket command updates, Secure Enclave | Approve, Reject, Request Changes. No workflow logic. |
| **Infrastructure Monitoring** | Detailed telemetry drill-down: hardware metrics, service health, model VRAM, agent status, error logs. | P0 | REST infrastructure endpoints, WebSocket telemetry | Read-only visualization. All metrics from workstation. |
| **Notifications** | Pager for critical alerts, job completions, approval requests, security events, and failures. | P0 | Push notifications, REST alerts endpoint | Display only. No notification generation. |
| **Projects** | List existing workstation projects and switch active context. | P1 | REST projects endpoint | Read-only list. No indexing. No local storage. |
| **Upload Center** | Send voice, images, PDFs, documents, and URLs to the workstation. | P1 | REST upload endpoint, multipart form data | Immediate upload. Never process locally. |
| **Settings** | Device configuration: pairing, workstations, notifications, theme, preferred model, VPN status. | P0 | REST settings endpoint, Secure Storage | Synchronized with workstation. |

## 2. Functional Decomposition

### Subsystem 1: AI Ops Telemetry (Executive Dashboard + Infrastructure Monitoring)
*   **System**: AegisOS Mobile App
*   **Subsystem**: Observability
*   **Feature**: Real-Time Hardware & Service Metrics
    *   **Sub-feature**: Health Score Gauge (overall system health rating).
        *   *Action*: Tap to navigate to Infrastructure Monitoring drill-down.
    *   **Sub-feature**: Resource Status Tiles (CPU, Memory, GPU).
        *   *Action*: Display current load percentage and temperature.
    *   **Sub-feature**: Service Health Cards (Ollama, LiteLLM, OpenClaw, Docker).
        *   *Action*: Tap to view detailed service status and port information.
    *   **Sub-feature**: Active Models and Agents summary.
        *   *Action*: Tap to view detailed model VRAM allocation or agent registry.

### Subsystem 2: AI Executive Chat
*   **System**: AegisOS Mobile App
*   **Subsystem**: Communication
*   **Feature**: Workstation Message Relay
    *   **Sub-feature**: Chat input with message forwarding.
        *   *Action*: Type message → forward to workstation → stream response tokens back.
    *   **Sub-feature**: Plan review cards (when workstation AI proposes multi-step actions).
        *   *Action*: Tap to review plan → navigate to approval gate.

### Subsystem 3: Human-in-the-Loop (HITL) Approvals
*   **System**: AegisOS Mobile App
*   **Subsystem**: Security & Governance
*   **Feature**: Cryptographic Approval Swipe
    *   **Sub-feature**: Command approval card (showing command type, risk level, payload).
        *   *Action*: Slide slider right to sign action with biometric key and resume workstation execution.
        *   *Action*: Tap reject to return failure to the workstation.
        *   *Action*: Tap request changes to send feedback text to the workstation.

### Subsystem 4: Notifications
*   **System**: AegisOS Mobile App
*   **Subsystem**: Event Display
*   **Feature**: Push Notification Feed
    *   **Sub-feature**: Alert cards categorized by severity (critical, warning, info).
        *   *Action*: Tap to view alert details.
    *   **Sub-feature**: Job completion notifications.
        *   *Action*: Tap to view job result summary.

### Subsystem 5: Projects
*   **System**: AegisOS Mobile App
*   **Subsystem**: Context Management
*   **Feature**: Project Selector
    *   **Sub-feature**: List of workstation projects.
        *   *Action*: Tap to set active project context for chat and commands.

### Subsystem 6: Upload Center
*   **System**: AegisOS Mobile App
*   **Subsystem**: File Transfer
*   **Feature**: Multimodal Upload
    *   **Sub-feature**: File picker (voice, images, PDFs, documents).
        *   *Action*: Select file → upload to workstation immediately.
    *   **Sub-feature**: URL input.
        *   *Action*: Enter URL → send to workstation for processing.
